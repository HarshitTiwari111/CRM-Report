# Deployment Guide

## 1. Push to GitHub

```bash
git remote add origin <your-github-repo-url>
git branch -M main
git push -u origin main
```

## 2. Database (already done)

MongoDB Atlas cluster is set up. Keep the connection string handy — you'll paste it into Render as `MONGO_URI`.

## 3. Deploy on Render

This repo includes a `render.yaml` Blueprint defining two services:
- `crm-report-api` — the Express backend (Node web service)
- `crm-report-client` — the React frontend (static site)

Steps:
1. Go to the Render dashboard → **New** → **Blueprint** → connect this GitHub repo. Render will read `render.yaml` and create both services.
2. On the **crm-report-api** service, set these environment variables (marked `sync: false` in the blueprint, so Render will prompt for them):
   - `MONGO_URI` — your Atlas connection string, with a database name in the path, e.g. `mongodb+srv://user:pass@cluster0.xxxx.mongodb.net/crm-report?appName=Cluster0`
   - `CLIENT_URL` — leave blank for now, come back after step 4
   - `SMTP_HOST` / `SMTP_PORT` / `SMTP_USER` / `SMTP_PASS` / `SMTP_FROM` — optional; if left blank, password-reset emails are just logged to the server console instead of sent
   - `JWT_SECRET` / `JWT_REFRESH_SECRET` are auto-generated by Render (`generateValue: true`) — no action needed
3. Deploy the API service. Once live, copy its URL, e.g. `https://crm-report-api.onrender.com`.
4. On the **crm-report-client** service, set:
   - `VITE_API_URL` = `https://crm-report-api.onrender.com/api` (the API URL from step 3, with `/api` appended)
   Deploy/redeploy the client service. Copy its URL, e.g. `https://crm-report-client.onrender.com`.
5. Go back to **crm-report-api** and set `CLIENT_URL` = `https://crm-report-client.onrender.com` (the client URL from step 4), then redeploy the API service so CORS allows requests from the deployed frontend.

The API's start command (`node src/utils/seed.js && node src/server.js`) seeds a superadmin (`admin@crm.local` / `Admin@123`) and default departments on first boot — **change this password immediately after first login in production.** The seed script is idempotent, so it's safe to run on every restart.

## 4. Verify

- Visit the client URL, log in with the seeded superadmin credentials, and confirm the dashboard loads.
- Check the API service logs in Render for `MongoDB connected` on startup.
